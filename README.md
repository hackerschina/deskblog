# GhostOsintWails-Markdown 0.5.2 更新与优化说明

## 版本概述

本次 0.5.2 版本重点围绕 `GhostOsintWails-Markdown` Markdown 文档管理与写作功能进行增强，优化了大量 Markdown 文件场景下的性能表现，并改进了文件夹、文件树、分类栏、写作体验和自动保存逻辑。

该版本适合用于本地 Markdown 文档管理、日记写作、资料整理、文件夹级 Markdown 浏览与编辑。

## 新增功能

### Markdown 文档管理

- 支持导入单个 `.md` / `.markdown` 文件。
- 支持导入 Markdown 文件夹。
- 支持递归读取文件夹内 Markdown 文档。
- 支持编辑导入文件夹内的 Markdown 文件内容。
- 支持将新建内容保存为本地 `.md` 文件。
- 支持写回已绑定的原始 Markdown 文件。

### 文件夹与文件树

- 增加最近打开文件夹记录。
- 最近文件夹数量不再限制为 8 个。
- 文件夹列表支持纵向滚动，文件夹较多时可像浏览器滚动条一样下拉浏览。
- 支持移除最近文件夹记录。
- 移除文件夹时只移除程序索引，不删除磁盘上的真实文件夹和文件。
- 增加 Markdown 文件树视图。
- 文件树默认折叠文件夹，展开后再渲染子内容。
- 文件树大量节点时分批显示，降低一次性渲染压力。

### 写作体验

- 支持 Markdown 编辑。
- 支持实时预览。
- 支持编辑、分屏、预览三种写作模式。
- 新建日记/文章时进入大页面写作界面。
- 支持 Markdown 图片预览。
- 支持粘贴图片并自动生成 Markdown 图片语法。
- 已绑定本地 Markdown 文件时，粘贴图片会保存到同目录 `attachments` 文件夹。
- 未绑定本地 Markdown 文件时，图片会以内嵌 data URL 方式插入正文。

### 自动保存

- 写作页面新增自动保存设置。
- 默认使用手动保存。
- 可选择 30 秒、1 分钟、3 分钟、5 分钟自动保存。
- 已绑定 `.md` 文件时，自动保存会写回原始文件。
- 未绑定文件时，自动保存到程序本地索引。

### 分类与搜索

- 支持新增、编辑、删除分类。
- 分类栏改为 `全部` 固定显示。
- 分类列表支持左右箭头滑动。
- 长分类名称自动截断，避免撑爆布局。
- 鼠标悬停分类按钮可查看完整分类名称。
- 搜索支持标题、路径、分类、标签和已加载正文内容。

### 数据仪表盘

- 增加 Markdown 文档统计。
- 显示文档总量。
- 显示今日新增数量。
- 显示近 7 天新增数量。
- 显示分类数量。
- 显示最近更新时间。
- 对大量 Markdown 文件场景，统计逻辑改为优先使用轻量元数据。

## 性能优化

### 大量 Markdown 文件优化

针对几百到几千个 Markdown 文件时软件卡顿的问题，本版本进行了重点优化：

- 导入/刷新文件夹时不再一次性读取所有 Markdown 正文。
- 新增轻量元数据扫描逻辑，只读取路径、文件名、大小、更新时间等信息。
- 文件树和文档列表只依赖轻量元数据渲染。
- 点击具体 Markdown 文件时，才按需读取该文件正文。
- IndexedDB 写入改为批量事务，减少大量文件导入时的数据库写入开销。
- 启动时会压缩旧版本已缓存的 Markdown 正文，降低后续内存占用。

### 写作性能优化

- Markdown 实时预览增加防抖，避免每输入一个字符就全量渲染。
- 编辑模式下不计算预览 HTML，只有分屏/预览模式才渲染。
- 附件扫描增加防抖，减少大文档输入时的全文正则扫描。
- 写作页面打开时暂停文件夹自动刷新，避免后台扫描影响输入体验。
- 自动刷新逻辑减少重复定时器重建。

### 文件树性能优化

- 文件树默认折叠，展开后才渲染子节点。
- 每层节点分批显示，降低超大目录一次性 DOM 渲染压力。
- 文件夹切换时增加真实加载状态，避免用户误以为软件卡死。

## 交互优化

- 文件夹加载时显示旋转 loading。
- 文件树加载时显示骨架屏。
- 刷新按钮加载时显示“加载中...”。
- 增加请求保护，连续切换文件夹时，旧请求不会覆盖新请求状态。
- 自动刷新在手动加载时暂停，避免打断当前操作。
- 原生确认框替换为应用内弹窗，标题显示为 `GhostOsint提示您`。
- 文件夹移除按钮已收进文件夹行内部，避免超出边界。

## 修复内容

- 修复选择应用/选择文件夹/手动添加路径导致白屏的问题。
- 修复 Wails 原生提示标题显示为 `wails.localhost` 的问题。
- 修复分类过多时后续分类不可见的问题。
- 修复分类横向滚动不明显的问题，改为左右箭头按钮控制。
- 修复大量 Markdown 文件导入后界面明显卡顿的问题。
- 修复性能优化后点击文件提示“读取 Markdown 正文失败”的问题。

## 当前注意事项

- 未打开过的 Markdown 文件默认只进入轻量索引，正文不会立即进入全文搜索。
- 点击打开某个 Markdown 文件后，正文才会被读取并进入可编辑状态。
- 特别大的 Markdown 文件仍建议关闭实时预览，使用手动预览或分屏预览。
- 粘贴大量图片前，建议先将文档保存为 `.md` 文件，避免正文内嵌过长 data URL。
